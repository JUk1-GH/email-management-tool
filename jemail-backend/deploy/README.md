# 部署脚本目录

这个目录保留的是项目自带的部署脚本和模板文件。

公开源码仓库不再展开任何具体平台、供应商或个人服务器方案；如果你需要使用这些脚本，请按自己的环境自行调整。

## 目录内容

- `provision-ubuntu-ec2.sh`：在远端环境安装依赖并写入服务配置
- `push-to-ec2.sh`：从本地同步代码到远端并执行 provision
- `jemail-backend.service`：服务模板
- `nginx-api.conf`：反向代理模板

## 提示

- 这些脚本主要是给熟悉自己环境的人做自动化用
- 如果你只是阅读、分享或二次开发源码，可以直接忽略这个目录
